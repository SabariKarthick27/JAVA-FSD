CSS3 Solution Guide

Welcome to the styling support center. This guide outlines how CSS3 variables, responsive media parameters, and alignment behaviors are resolved in this project.

1. How do we test mobile screen responsiveness?

The viewport layout automatically pivots under 768px. Our media query re-orders navigation menus vertically, resizes typography scales, stacks columns into one, and stretches form containers. You can simulate mobile layouts by toggling the device option in Chrome DevTools.

2. Where are CSS3 variables declared?

Branding variables are declared in the root scope selector :root inside the external sheet styles.css. This allows clean, systematic editing of text colors, theme shadows, HSL values, and borders across all HTML pages.

3. What is the layout difference between "hidden" and "none"?

In our layout sandbox section on the homepage, clicking the visibility button toggles visibility: hidden. Notice that while the element disappears, it continues to take up physical container space. Clicking the display button toggles display: none, which collapses the element entirely so adjacent elements shift to fill the gap.

4. Are the tables collapsed dynamically?

Yes. The schedule table leverages border-collapse: collapse; alongside custom table padding, centered text aligns, and alternate-row zebra striping (`tr:nth-child(even)`), producing an exceptionally tidy layout.
